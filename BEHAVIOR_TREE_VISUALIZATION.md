我已將 Web 介面的行為樹視覺化更新為使用**傳統行為樹標準符號**，符合學術界和工業界常用的表示方式。

## 🎨 節點形狀說明

### 1. **Sequence（序列節點）** - 矩形 ▭
```
┌─────────────┐
│  Sequence   │
│  按順序執行  │
└─────────────┘
```
- **形狀**: 矩形（Rectangle）
- **顏色**: 藍色 (#3b82f6)
- **功能**: 按順序執行所有子節點，全部成功才算成功
- **失敗條件**: 任一子節點失敗則整個序列失敗

### 2. **Selector（選擇器節點）** - 菱形 ◇
```
    ◇
   ╱ ╲
  ╱   ╲
 ╱ Sel ╲
╱  ect  ╲
╲  or   ╱
 ╲     ╱
  ╲   ╱
   ╲ ╱
    ◇
```
- **形狀**: 菱形（Diamond）
- **顏色**: 橘色 (#f59e0b)
- **功能**: Fallback/Priority selector，依序嘗試子節點直到有一個成功
- **成功條件**: 任一子節點成功則選擇器成功

### 3. **Parallel（並行節點）** - 平行四邊形 ▱
```
  ┌────────────┐
 ╱  Parallel   ╱
╱─────────────╱
```
- **形狀**: 平行四邊形（Parallelogram）
- **顏色**: 紫色 (#8b5cf6)
- **功能**: 同時執行多個子節點
- **應用**: 需要並發執行的任務

### 4. **Action/Condition（行為/條件節點）** - 橢圓 ⬭
```
  ╭──────────╮
 │   Action   │
 │   行為節點  │
  ╰──────────╯
```
- **形狀**: 橢圓（Ellipse）
- **顏色**: 綠色 (#10b981)
- **功能**: 葉節點，執行具體的動作或檢查條件
- **範例**: `PickUp`, `PlaceDown`, `DetectObjects`

---

## 🎯 執行狀態顏色

行為樹在執行時，節點會根據狀態改變顏色（覆蓋原始類型顏色）：

| 狀態 | 顏色 | 說明 | 視覺效果 |
|------|------|------|----------|
| ✅ **Success** | 綠色 | 節點執行成功 | 綠色光暈 |
| 🔄 **Running** | 亮藍色 | 節點正在執行 | 脈衝動畫 + 藍色光暈 |
| ❌ **Failure** | 紅色 | 節點執行失敗 | 紅色光暈 |
| ⚠️ **Invalid** | 灰色 | 節點無效或未初始化 | 半透明 |

## 📐 視覺化特性

### 連線樣式
- **普通連線**: 灰色實線
- **執行中連線**: 橘色虛線動畫（stroke-dasharray）
- **成功連線**: 綠色粗線
- **失敗連線**: 紅色粗線

### 互動功能
- **點擊節點**: 查看節點詳細資訊
- **滑鼠懸停**: 節點邊框加粗，顯示提示訊息
- **拖曳畫布**: 移動整個樹狀圖
- **縮放**: 使用滑鼠滾輪縮放視圖

### 控制按鈕
- **展開**: 展開所有節點
- **收縮**: 收合所有子節點
- **置中**: 將樹狀圖置中顯示

## 🚀 使用方式

1. 啟動 Web 介面：
   ```bash
   rosrun niryo_web_interface web_camera_node.py
   ```

2. 開啟瀏覽器訪問：
   ```
   http://10.10.10.10:5000
   ```

3. 發送語音指令或使用 ROS 服務生成行為樹：
   ```bash
   rosservice call /generate_behavior_tree "task_description: '把藍色方塊拿起來放到白色區域'"
   ```

4. 在網頁右側的「行為樹狀態」區域即可看到使用傳統符號的視覺化行為樹！
